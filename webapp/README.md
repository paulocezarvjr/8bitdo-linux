# 8BitDo Keyboard Configurator (WebHID)

A browser-based configurator for the 8BitDo Retro Mechanical Keyboard — the same
read/remap protocol as `tools/kbd_read.py` / `kbd_write.py`, but running entirely
in the browser via the **WebHID API** (the same approach as VIA / NuPhy Console).
No install, no Ultimate Software, no Windows.

## Run it

WebHID needs a **secure context**, which includes `http://localhost`. Serve the
folder and open it in a Chromium browser (Chrome/Edge):

```sh
cd webapp
python3 -m http.server 8000
# then open http://localhost:8000 in Chrome or Edge
```

1. Connect the keyboard by **USB cable** with the power switch **OFF** (config mode).
2. Click **Connect keyboard** and pick the *8BitDo Retro Keyboard* entry.
3. The current profile + remaps load. Use **Remap a key** to change one, **reset**
   to restore a key's default, or **Disable key**.
4. Remaps are written to the on-keyboard **profile** — press the **8BitDo key** on
   the keyboard to activate that profile and see them live.

> Don't run the Python tools at the same time — both open the same HID device.

## Files

- `index.html` / `style.css` — UI (retro 8-bit theme).
- `kbd.js` — WebHID driver (protocol: report `0x52` out / `0x54` in, Usage Page `0x008C`).
- `app.js` — UI wiring.
- `keys.js` — key tables, auto-generated by `../tools/gen_keys_js.py` from the
  goncalor submodule. Regenerate with `python3 tools/gen_keys_js.py`.

## Browser support

Chromium-based only (Chrome, Edge, Brave, Opera). Firefox/Safari do not implement
WebHID. The protected-usage blocklist does not affect us: configuration runs on
the vendor collection (Usage Page `0x008C`), not the keyboard collection.

## Status

Keyboard read + remap implemented. The protocol is verified against hardware via
the Python tools; the WebHID glue should be tested in-browser against the device.
